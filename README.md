# VPN Site Manager

Интерактивный установщик и менеджер production-развёртывания сайта `dorillo/vpn-site` на чистой Ubuntu 24.04. Скрипт устанавливает системные зависимости, собирает Python 3.14, настраивает PostgreSQL, systemd, Nginx, Let's Encrypt, резервное копирование и при необходимости UFW.

Поддерживаются только Ubuntu 24.04 на `amd64` и `arm64`. Для установки требуются не менее 1 ГиБ RAM и 4 ГиБ свободного места. Скрипт рассчитан на отдельный сервер без другого сайта на портах 80 и 443.

## Перед установкой

Подготовьте:

- домен с публичной записью A и/или AAAA, указывающей на сервер;
- открытые извне TCP-порты 80 и 443;
- SMTP-сервер для кодов входа;
- URL и токен Remnawave;
- email первого администратора;
- fine-grained GitHub token с доступом только к приватному репозиторию сайта и разрешением `Contents: Read-only`.

Если включаете UFW, заранее выясните фактический TCP-порт SSH. Ошибочный порт может закрыть удалённый доступ. YooKassa при первой установке необязательна.

## Установка

Запустите на сервере от пользователя с `sudo`:

```bash
curl -fsSL https://raw.githubusercontent.com/dorillo/vpn-site-manager/main/install.sh | sudo bash -s -- install
```

Установщик обновит пакеты Ubuntu и запросит все необходимые значения. Сборка Python 3.14 из исходников может занять несколько минут. После установки команда управления доступна как:

```bash
sudo vpn-site
```

GitHub-токен используется только в памяти текущего процесса и очищается после операции. Он не записывается в env-файл, конфигурацию менеджера, URL команды или логи. При каждом обновлении приватного сайта токен запрашивается снова.

## Команды

| Команда | Назначение |
| --- | --- |
| `sudo vpn-site` | Интерактивное меню |
| `sudo vpn-site install` | Мастер первоначальной установки |
| `sudo vpn-site status` | Состояние сервисов и локальная health-проверка |
| `sudo vpn-site start` | Запуск backend |
| `sudo vpn-site stop` | Остановка backend; Nginx остаётся запущен |
| `sudo vpn-site restart` | Перезапуск с health-проверкой |
| `sudo vpn-site update` | Обновление сайта из приватного GitHub-репозитория |
| `sudo vpn-site backup` | Неинтерактивный бэкап PostgreSQL и конфигурации |
| `sudo vpn-site diagnose` | Проверка сервисов, env, HTTPS, сертификата и таймера бэкапов |
| `sudo vpn-site logs` | Последние 200 строк журнала приложения и ошибки Nginx |
| `sudo vpn-site update-manager` | Обновление менеджера из публичного репозитория |
| `sudo vpn-site os-update` | Обновление пакетов Ubuntu |
| `sudo vpn-site repair` | Восстановление управляемых файлов systemd, Nginx и Certbot |

Команды `status`, `start`, `stop`, `restart`, `backup`, `diagnose`, `logs`, `os-update` и `repair` можно вызывать без интерактивного терминала, например из системы мониторинга. Установка, меню и обновление приватного сайта требуют TTY.

## Возможности меню

В разделе настроек окружения можно менять публичный режим и `PUBLIC_COPY_MODE`, бренд, заголовки, ИНН, SMTP, Remnawave, YooKassa, лимиты и email первого администратора. Полный env-файл также можно открыть в `${SUDO_EDITOR:-${EDITOR:-nano}}`; перед применением создаётся резервная копия, конфигурация валидируется кодом сайта, а сервис проходит health-проверку. При ошибке или прерывании восстанавливается предыдущий файл.

Раздел сертификатов показывает срок действия, выполняет тестовое или принудительное продление и переносит сайт на другой домен. Сертификаты выдаются через Let's Encrypt webroot. Автопродление выполняет системный `certbot.timer`, после успешного продления Nginx проверяется и перезагружается.

Раздел бэкапов создаёт, показывает и восстанавливает дампы базы, а также позволяет изменить срок хранения от 1 до 3650 дней. По умолчанию бэкапы хранятся 14 дней.

## Обновления и откат

Каждая версия сайта разворачивается в отдельный каталог. Перед обновлением создаётся проверенный custom-format дамп PostgreSQL и архив конфигурации. Миграции Alembic выполняются до переключения симлинка. Если миграция, запуск или health-проверка завершаются ошибкой либо процесс прерывается, менеджер возвращает прежнюю базу, release, Git ref и исходное состояние сервиса (запущен или остановлен).

Хранятся три последние версии сайта и менеджера. Резервные копии создаются ежедневно через systemd timer около 03:20 с случайной задержкой до 30 минут и также перед каждым обновлением или восстановлением базы. Восстановление выбранной базы сначала создаёт страховочный бэкап и автоматически возвращает его, если выбранный архив не восстановился.

Архив конфигурации предназначен для аварийного ручного восстановления и содержит секреты. Автоматический пункт меню восстанавливает только PostgreSQL. Перед переносом бэкапов во внешнее хранилище используйте шифрование.

## YooKassa

Интеграцию можно пропустить при установке и включить позднее в `Environment settings -> YooKassa`. Менеджер запросит shop ID и secret key, сгенерирует отдельный webhook token и покажет URL вида:

```text
https://example.com/payments/yookassa/webhook?token=GENERATED_TOKEN
```

Этот URL необходимо зарегистрировать в кабинете YooKassa. При смене домена `YOOKASSA_RETURN_URL` обновляется автоматически, но webhook в кабинете YooKassa нужно изменить вручную.

## Размещение файлов

| Путь | Содержимое |
| --- | --- |
| `/opt/vpn-site/releases/<sha>` | Неизменяемые версии сайта и virtualenv |
| `/opt/vpn-site/current` | Активная версия сайта |
| `/opt/vpn-site-manager/releases/*` | Версии менеджера |
| `/opt/vpn-site-manager/current` | Активная версия менеджера |
| `/etc/vpn-site/vpn-site.env` | Секреты и настройки приложения, `root:vpn-site`, mode `0640` |
| `/etc/vpn-site/manager.conf` | Состояние менеджера, `root:vpn-site`, mode `0640` |
| `/var/backups/vpn-site/database` | Дампы PostgreSQL |
| `/var/backups/vpn-site/config` | Архивы конфигурации и копии env |
| `/etc/nginx/sites-available/vpn-site` | Управляемая конфигурация Nginx |
| `/etc/systemd/system/vpn-site*` | Сервис приложения и таймер бэкапов |

## Безопасность и ограничения

- Запускайте менеджер только от root; одновременные операции блокируются через `flock`.
- Env-файл содержит пароль БД, SMTP, Remnawave, YooKassa и ключ сессий. Не публикуйте его и не добавляйте в Git.
- Менеджер обновляет всю ОС через `apt-get full-upgrade`; если `/var/run/reboot-required` существует, перезагрузку нужно выполнить в согласованное окно самостоятельно.
- UFW управляется только когда вы явно включили его в мастере установки. Менеджер не удаляет сторонние правила.
- Скрипт проверяет наличие DNS-записи, но не может достоверно подтвердить изнутри сервера правильность NAT, маршрутизации и внешнего firewall.
- Полностью автоматического удаления сайта нет: это сознательно исключает случайное удаление production-данных.

## Разработка

Локальные проверки не изменяют систему:

```bash
bash tests/run.sh
```

Они проверяют синтаксис Bash, валидаторы, рендер Nginx и Python-инструменты env-файла. Полный интеграционный тест установки требует отдельной чистой VM с Ubuntu 24.04, рабочим DNS, PostgreSQL, systemd и тестовым приватным репозиторием.
